# Naming Conventions (All Languages)

## Universal Rules
- Descriptive over clever
- Full words, not abbreviations (except standard ones)
- Boolean variables: `is`, `has`, `should` prefixes
- Array/lists: plural nouns
- Avoid negatives in boolean names

## Language-Specific
### TypeScript/JavaScript
- camelCase for variables/functions
- PascalCase for classes/components
- SCREAMING_SNAKE for constants

### Python
- snake_case for variables/functions
- PascalCase for classes
- SCREAMING_SNAKE for constants

### Go
- PascalCase for exported names
- camelCase for unexported names
